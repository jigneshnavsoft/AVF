Research: Week 1 Discussion

What types of native features (on Android and iOS devices) would you be able to implement with the use of HTML/CSS/JS only?

According to the PhoneGap chart I can access the iOS and Android Accelerometer, Camera, Contacts, File, GeoLocation, Media, Network and Notification (sound, vibration, alert), and the Local Storage.

For your CRUD app revision and/or Demo App components, which of these do you think you will implement?

I want to use the Local Storage, Camera, and Contacts for my next CRUD revision. I might also add Local Storage to my Demo App.

Briefly describe some past 'lessons learned' regarding workflow as it relates to research, design, development, etc.

I learned that I have to start working on my weekly school assignments very early and I also need to plan to work very long & late hours to fit everything into my schedule. My phone is off, I don't answer the door, I don't go anywhere so I can just focus on getting all my work done.

Please describe any design intentions you may have for the AVF Demo app and the revised/new version of your CRUD app.

I want both of my apps to continue with the blue deep background. I might bring back my original icons to be shown in the listviews and represent the project type for the list items.

If not covered above, please also describe any functionality intentions you may have for both the AVF Demo app and the revised/new version of your CRUD app.

I plan to get away from the couch db because it causes alot of issues regarding internet speed and I've even seen error messages that are on the cloudant website server side that I have no control over to fix. My app uses such a small amount of data for storage / creation that my end users' local storage will be able to handle it fine.
